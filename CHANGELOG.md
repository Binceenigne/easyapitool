# 更新日志

## 1.0.7 - 2026-07-17

- 修复单文件 EXE重启时复用旧 PyInstaller 临时目录，导致内嵌页面丢失和 `_MEI` 目录清理警告的问题。
- 极简标题栏调整为 22px，比默认标题栏缩小约 33%，并适当增大标题栏图标和窗口按钮。
- 极简模式的最小窗口尺寸调整为 220×96，默认和 Windows 原生模式仍保持 260×120。

## 1.0.6 - 2026-07-17

- 设置页新增默认、极简和 Windows 原生三种标题栏样式，切换后可直接重启应用生效。
- 极简标题栏移除应用标题，将密钥选择、刷新、添加、删除和设置收纳为小图标，并将高度缩小约 50%。
- Windows 原生标题栏恢复系统窗口边框与原生拖拽、缩放，同时保留原有密钥工具栏和关闭行为设置。

## 1.0.5 - 2026-07-16

- 下载更新进度改为完整可见的 4×4 点阵块，并按用量进度条相同的方式从左向右推进。
- 修复应用退出过快时更新脚本因原进程已不存在而中止、导致下载完成后没有继续安装的问题。
- 更新脚本增加启动握手、文件替换重试和错误日志；只有确认外部更新器已运行后才退出当前应用。
- 下载并校验完成后由用户选择稍后重启或立刻重启，更新日志不再展示 SHA-256 校验值。
- 无限账户不再根据动态余额推导额度上限；只有上游明确返回的总额度和 5h / 1d / 7d 限制发生变化时才发送通知。

## 1.0.4 - 2026-07-16

- 每次进入应用时检查更新，发现新版本后通过模态框展示版本、更新日志、下载进度和确认操作。
- 支持暂时拒绝更新或忽略指定版本；手动检查更新时仍可重新查看已忽略的版本。
- 用量趋势图新增 1h / 10min 区间切换，10min 模式展示最近 2 小时的 12 个滚动区间。
- 修复正常额度消费被误报为“总额度上限调整”的问题，只在真实额度上限或 5h / 1d / 7d 限额变化时通知。
- 应用与托盘图标使用紫色结形标识；Windows 通知使用独立的芯片心电图像素图标。

## 1.0.3 - 2026-07-16

- 修复自动更新在连接 GitHub 下载源时可能长时间停留在“正在下载更新”的问题。
- 提醒阈值保存后以后端实际值为准，Windows 通知标题映射用户自定义阈值。
- 额度上限或 5h / 1d / 7d 速率限制发生提高、降低、新增或取消时发送 Windows 通知。
- 通知与应用图标改为更简洁的芯片心跳线像素风格。

## 1.0.2 - 2026-07-16

- 默认余量报警阈值调整为 50%、25% 和 10%。
- 更新进度条改为 4×4 点阵随机填充，空闲时保持熄灭。
- 更新日志改用更清晰的大字号中文 Markdown 排版。
- Windows 通知按正常、一级、二级和严重状态使用高清点阵机能风图标。

## 1.0.1 - 2026-07-15

- 修复频率窗口已用额度略高于上限时，可用额度被重复计算为负数的问题。
- 频率窗口现在优先采用上游返回的剩余额度，并统一钳制在 `0` 到窗口上限之间。
- 超额时仍保留实际已用额度展示，但剩余比例最低为 0%、已用比例最高为 100%。

## 1.0.0 - 2026-07-15

- 新增 GitHub Release 自动更新，下载进度使用自适应点阵进度条。
- 新增每次启动、每周或手动检查更新频率设置。
- 新增关闭应用、最小化到托盘或每次询问的关闭按钮行为。
- 新增 Windows 开机自启设置。
- 新增应用内更新日志展示。
- 修复点阵进度条在不同窗口宽度下越过父组件的问题。
- 优化点阵渲染与关键动态节点的 GPU 合成，减少重复 DOM 重建。
- 为内部模态框增加平滑进退场动画，更新下载点阵使用 6 秒长缓动连续展示进度。
